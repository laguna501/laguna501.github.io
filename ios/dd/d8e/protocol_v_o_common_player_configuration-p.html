- (VO_OSMP_RETURN_CODE) enableLiveStreamingDVRPosition: (bool)  enable

Enable defining position based on live streaming DVR window.

The default is enabled.

When enabled:

  • VOCommonPlayerStatus#getPosition returns 0 when playing at the live head. A negative position represents the offset into the DVR window with respect to live head
  • VOCommonPlayerStatus#getMaxPosition always returns 0 which represent the live head
  • VOCommonPlayerStatus#getMinPosition returns (getMaxPosition - DVR window size)

When disabled:

  • VOCommonPlayerStatus#getPosition returns the offset with respect to the beginning of the playback
  • VOCommonPlayerStatus#getMaxPosition returns the offset between the live head and the beginning of the playback
  • VOCommonPlayerStatus#getMinPosition returns (getMaxPosition - DVR window size)
Parameters
value[in] Enable/Disable; true to enable(default), false to disable.
Returns
VO_OSMP_ERR_NONE if successful

- (VO_OSMP_RETURN_CODE) setBitrateThreshold: (int)  upper
lower: (int)  lower 

Set upper/lower bitrate threshold for adaptation.

Parameters
upper[in] Maximum bitrate that the adaptation must not exceed (bps) in addition to the constraint defined in the device capability file VOCommonPlayerControl#setDeviceCapabilityByFile.
  • If the value is 0, the maximum bitrate is constrained by the device capability file if it is specified. Otherwise the maximum bitrate is unconstrained.
  • If the value is not 0 and the device capability file is not specified, then it defines the maximum bitrate threshold
  • If the value is not 0 and the device capability file is specified, then the maximum bitrate threshold is defined as min(upper, max bitrate in device capability file)
Parameters
lower[in] Mimimum bitrate that the adaptation must not drop below (bps).
  • If the value is 0, then the minimum threshold is unconstrained
  • If the value is not 0, then it defined the minimum bitrate threshold that adaptation must not drop below.
Returns
VO_OSMP_ERR_NONE if successful.
 This API should be called before calling the VOCommonPlayerControl#open:flag:sourceType:openParam: method.

- (VO_OSMP_RETURN_CODE) setInitialBitrate: (int)  bitrate

Set the initial bitrate.

The player will select the closest stream at or below that number. If none is available, it will pick the closest one above. If an initial bitrate is not set, the player will follow the streaming protocol specification. If the behavior is undefined in the protocol being used, the player will pick a stream based on cap.xml, the order in the manifest, and other heuristics.

Parameters
bitrate[in] Bitrate <bps>.
Returns
VO_OSMP_ERR_NONE if successful
This API should be called before calling the VOCommonPlayerControl#open:flag:sourceType:openParam: method.

- (VO_OSMP_RETURN_CODE) setZoomMode: (VO_OSMP_ZOOM_MODE)  mode
rect: (Rect)  rect 

Show a portion of video.

Parameters
mode[in] Zoom mode. Refer to VO_OSMP_ZOOM_MODE.
rect[in] The rectangular area of the video to be displayed. This argument is only used when the zoom mode is VO_OSMP_ZOOM_ZOOMIN. The rectangular area is expressed in terms of video pixel dimensions. When the video size is changed (e.g. due to a dynamic resolution switching), the player will automatically adjust the dimensions of the displayed rectangle according to the video scaling factor.

This API currently is not implemented on Mac OS.

Returns
VO_OSMP_ERR_NONE if successful
